# Design Direction

Use this skill when the goal is to improve how an interface looks and feels, not just how it functions.

## Quick Start

1. Read the existing UI first and identify what visual language is already present.
2. Decide whether to preserve the current style or introduce a stronger new direction.
3. Improve hierarchy before decoration: layout, spacing, typography, contrast, and CTA emphasis.
4. Apply a coherent visual system across the touched surface instead of isolated tweaks.
5. Verify desktop and mobile results before closing the task.

## Workflow

### 1. Establish the direction

- Identify the brand mood: editorial, premium, playful, minimal, utilitarian, etc.
- Pick 2-4 visual anchors: typography, color, shape language, density, motion, texture.
- If the product already has a recognizable style, extend it instead of replacing it.

### 2. Improve the core visual hierarchy

- Make the primary action obvious within one screen.
- Separate headline, supporting copy, metadata, and controls clearly.
- Reduce visual noise before adding new embellishments.
- Use spacing to group related elements and create rhythm.

### 3. Strengthen the system

- Prefer reusable tokens or variables for colors, radius, shadows, spacing, and transitions.
- Keep buttons, cards, forms, and navigation visually related.
- Ensure hover, focus, active, and disabled states feel intentional.

### 4. Add personality carefully

- Use one strong typographic move rather than many competing styles.
- Use gradients, overlays, borders, or texture to add atmosphere when helpful.
- Avoid generic "AI template" layouts that could belong to any product.

## Guardrails

- Do not sacrifice readability for style.
- Do not introduce a new visual language in one component only.
- Do not overuse animation, blur, glass, or gradients without a clear role.
- Do not make mobile a compressed afterthought.
