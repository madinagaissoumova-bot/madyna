# UX Audit Checklist

## Flow

- Is the next step obvious?
- Does each page explain why the user is here?
- Can the user recover easily from a mistake?
- Are success and error states concrete and honest?

## Forms

- Labels are clear and specific
- Validation appears at the right moment
- The first invalid field is easy to find
- Autofill and input modes are used when relevant

## Navigation

- Current location is visible
- Mobile menu behavior is predictable
- Important destinations are reachable quickly
- Backtracking is easy

## Trust

- Buttons say what really happens
- Confirmation language matches actual system behavior
- Demo or local-only behavior is made explicit
- The interface does not overpromise
