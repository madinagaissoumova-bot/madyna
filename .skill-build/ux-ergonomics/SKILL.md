# UX Ergonomics

Use this skill when the problem is friction, confusion, hesitation, or unnecessary effort in the user journey.

## Quick Start

1. Identify the task the user is trying to complete.
2. Find where the flow becomes ambiguous, overloaded, or misleading.
3. Fix the highest-friction step first.
4. Improve labels, feedback, sequencing, defaults, and navigation cues before adding complexity.
5. Re-check the flow on mobile and keyboard/focus paths.

## Workflow

### 1. Map the user flow

- Define the intended path: discover, compare, decide, act, confirm.
- Note where the interface asks the user to think too much or guess the next step.
- Look for missing progress cues, misleading labels, and dead ends.

### 2. Reduce friction

- Make the next action explicit.
- Add helper text only where it removes uncertainty.
- Prefer progressive disclosure over dumping everything at once.
- Improve empty states, validation messages, and post-action feedback.

### 3. Strengthen ergonomics

- Ensure tap targets are comfortable on mobile.
- Keep critical actions reachable without excessive scrolling.
- Improve focus visibility and keyboard navigation where practical.
- Align UI wording with what the system really does.

### 4. Validate honestly

- Do not imply a flow is complete if the backend behavior is still a mock.
- Call out demo-only steps, local-only persistence, or fake confirmation states.
- Prefer clear expectations over polished but misleading UX.

## Guardrails

- Do not optimize aesthetics while leaving the flow confusing.
- Do not add more steps unless they remove bigger confusion.
- Do not hide critical information behind weak labels.
- Do not present local-only or fake actions as real account, payment, or order submission features.
